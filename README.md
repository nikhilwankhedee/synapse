## What is SYNAPSE?

Most Python codebases are black boxes at runtime.  
You run the code. Something breaks. You guess where.

**SYNAPSE changes that.**

It traces your program during execution and renders an interactive graph showing exactly how functions call each other ‚ÄĒ where time is spent, where errors originate, and how data flows through your system.

> One command. Your entire runtime, mapped.

## Usage

```bash
synapse path/to/your_project
```

That's it. SYNAPSE will:

1. **Trace** ‚ÄĒ instruments your program and records every function call at runtime
2. **Analyze** ‚ÄĒ maps relationships, frequencies, and error paths
3. **Render** ‚ÄĒ opens an interactive execution graph in your browser

## Scale Modes

SYNAPSE adapts to your project size automatically based on available system memory.

| Mode | Max Nodes | Best For |
|------|-----------|----------|
| `CPU SAFE` | ~1,200 | Scripts, small modules |
| `BALANCED` | ~3,000 | Mid-size applications |
| `LARGE GRAPH` | ~8,000 | Frameworks, large services |
| `FULL GRAPH` | ~50,000 | Monorepos, deep systems |

## Features

- **Runtime call tracing** ‚ÄĒ captures the real execution path, not a static estimate
- **Interactive graph** ‚ÄĒ zoom, pan, click, explore
- **Hotspot detection** ‚ÄĒ instantly see your most-called functions
- **Error highlighting** ‚ÄĒ trace exceptions back to their source
- **Code preview** ‚ÄĒ inspect function source without leaving the graph
- **Execution replay** ‚ÄĒ visualize the runtime flow in sequence
- **Auto-scaling** ‚ÄĒ intelligently adjusts render mode to your hardware


- **Zoom** into hot execution paths
- **Click** any function to read its source
- **Replay** the runtime to follow call sequences
- **Spot** yellow nodes ‚ÄĒ those are your performance candidates
- **Follow** red nodes back to error roots
